### 通用 Agentic RAG 智能问答系统

2026-01 ~ 2026-04

全栈开发

项目简述：针对传统 RAG 存在的语义割裂、上下文截断、跨文档对比难及长文本宏观总结弱等痛点，通过重构检索链路、引入意图路由与分层记忆机制，开发 Agentic RAG 系统

技术栈：Rag+LangChain4j + Spring Boot + Milvus + Elasticsearch + Redis + Kafka + Canal

- 父块回表与混合检索：设计智能双轨分块（父子块索引），子块精准向量召回后逆向查完整父块，解决上下文截断；双路召回（BM25 + HNSW）经 RRF 融合， Recall@10提升至0.82，上下文精确度提升30.8%。
- 高级推理与宏观检索：实现多跳迭代检索（类似 IRCoT），支持跨文档对比；离线构建文档摘要树（参考 RAPTOR），跨文档对比类问题的答案相关性从 0.52 提升至 0.71。
- 三级记忆与动态遗忘：L0 滑动窗口（Redis + GZIP 压缩）→ L1 渐进式摘要 → L2 实体记忆；短转长异步提炼用户偏好，基于访问频次与时间的权重衰减自动淘汰低活记忆，长对话场景下忠实度从 0.68 提升至 0.83。
- 意图路由与 Skill 加载：三级意图路由（正则→LLM→兜底）；Skill 三级动态加载（元数据→指令→资源），渐进式披露降低上下文 ,Token减少 14% 。
- 自动化评估与可观测：LLM-as-a-Judge围绕忠实度、上下文精确度、相关性自动打分；集成 LangSmith 全链路追踪，实现白盒化性能瓶颈定位。

### 多 Agent 电商推荐与营销系统

2026-02 ~ 2026-04

独立开发

项目简述：基于  LangChain 构建企业级多 Agent 协同系统，实现从用户理解、商品推荐到营销文案生成的全链路智能化，支持实时特征、A/B 测试与成本优化。

技术栈：LangGraph + LangChain + Milvus + Redis + MySQL

- Supervisor + 4 Agent 并行编排：基于 Supervisor 模式构建用户画像、商品推荐、营销文案、库存决策 4 个专业 Agent，设计两阶段并行架构，端到端延迟 P99 < 2s。
- 实时用户特征工程：针对用户行为秒级变化及跨会话偏好丢失问题，基于 Redis Sorted Set 存储行为序列，滑动窗口（1h/24h/7d）实时计算 RFM 模型与偏好标签，特征更新延迟 <100ms。
- 电商推荐检索：先多路召回（协同过滤 + Milvus 向量 + 热度），再结合用户实时特征与短期记忆，通过 LLM 对候选集精排，并引入 MMR 去重与库存/价格约束，点击率（CTR）从2.1% 升至 2.9%。
- A/B 测试与动态调优：用户 ID 一致性哈希分桶，Thompson Sampling（Beta 分布采样）自动分配流量，支持 Agent / 模型 / Prompt 三层实验。
